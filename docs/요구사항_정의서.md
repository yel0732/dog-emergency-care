# 구해줘 멍즈 요구사항 정의서

- 프로젝트 도메인: AI 기반 반려견 응급증상 분석 및 응급도 판단 지원 서비스

---

## 1. 프로젝트 개요

구해줘 멍즈는 반려견 보호자가 응급 상황에서 증상을 입력하고, 응급도 판단 결과와 행동 안내, 관련 응급처치 영상, 동물병원 정보, 음식 안전 정보를 한 흐름에서 확인할 수 있도록 지원하는 웹 서비스이다.

본 서비스는 LLM이 응급도를 직접 판단하는 방식이 아니라, 사용자의 자연어 증상 입력을 구조화하고 안내문을 이해하기 쉽게 다듬는 데 AI를 활용한다. 
실제 응급도 산정은 백엔드에서 근거 DB와 규칙 기반 로직을 바탕으로 수행되며, 최종 결과는 백엔드 검증 후 사용자에게 제공된다.

## 2. 사용자 및 권한

| 사용자 | 설명 | 주요 권한 |
|---|---|---|
| 비회원 | 로그인하지 않은 방문자 | 공개 정보 조회, 로그인/회원가입 진입 |
| 회원 | 반려견 보호자 | 반려견 관리, 응급 체크, 게시글/댓글, 찜, 팔로우, 병원/음식 검색 |
| 관리자 | 서비스 운영자 | 응급처치 영상 등록/수정/삭제, 게시글/댓글 삭제 |

## 8. 인증 및 예외 처리 정책

- 인증은 Spring Security와 JWT 기반으로 처리한다.
- 로그인 성공 시 Access Token을 저장하고, 보호 API 요청 시 `Authorization: Bearer <accessToken>` 헤더를 포함한다.
- 보호 API에서 401이 반환되면 로그인 필요 안내 후 로그인 화면으로 이동한다.
- 관리자 기능은 프론트엔드 버튼 숨김과 백엔드 권한 검증을 함께 적용한다.
- 400, 401, 403, 404, 500 및 네트워크 오류는 사용자 친화적인 문구와 함께 안내한다.
- 지도, 병원, AI 등 외부 API 실패 시 fallback 문구와 재시도 방법을 제공한다.
